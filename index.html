HTML

  • Hypertext Markup Language (HTML) uses elements to add content and structure webpage to ease navigation.
  • An element refers to the entire element from opening to closing tag (and all content in between tags), whereas a tag refers only to what appears inside the angle brackets.
  • Elements nested inside other elements are called child elements. The element that holds all the children is known as the parent element.
  • Elements can be semantic or non-semantic. A semantic HTML element clearly desribes the content it contains and make the webpage accessible and easy to read by other devs, users, and in different browsers (so use often). Semantic examples: body and section. Non-semantic examples: span and div.
  • Start HTML document with document type declaration followed by the HTML element.
  • The HTML element is the parent of two important child elements, head and body.
  • Content is added with text, header, image, and list elements. Page division and description is added with meta-related, content division, and semantic elements.

Head:

  • The head element contains information or metadata about the webpage. Important for accessibility, search engines, and performance. This is not visible to the user.
  • The title element contains the webpage title. Important for search engine optimization (SEO).
  • The style element defines the CSS styles.
  • The base element defines the base URL.
  • The link element connects an external resource. rel attribute defines relationship between documents. href attribute points to URL of external resource.
  • The meta element defines metadata such as character set, descriptions, keywords, author, and viewport.
  • The script element embeds client-side scripts.

Body:

  • The body element represents visible content shown to the user.
  • The header element appears at top of page and contains navigation menu or introductory content.
  • The main element contains the main content.
  • The footer element appears at bottom of page and contains author, copyright, sitemap, and navigation.
  • The span element creates an in-line division to markup part of text or document and is non-semantic.
  • The div element creates a division in the doucment and is non-semantic.
  • The h1, h2, h3, h4, h5, and h6 elements are text headings (listed in decreasing size).
  • The p element is used for a paragraph or block of text.
  • The ul, ol, and li elements stand for unordered list, ordered list, and list items.
  • The img element contains information of images displayed. src attribute defines location of image. alt attribute describes appearance and functionality of image, important for screen readers and improves search engine rankings.
  • The br element creates a line break or line of empty space.
  • The a element stands for anchor and creates links to webpages. href attribute points to the URL.
  • The section element creates sections of content (for organization). class attribute creates type of class for an element and multiple elements can use the same class. id attribute is used to identify elements and select them.

CSS

  • Cascading Style Sheets (CSS) gives the content style. It defines the colors, aesthetics, visual layout, and organization.
  • There are three ways to use CSS: in-line, internal style sheet, or external style sheet.
  • In-line is good for quick, permanent changes but is less flexible.
  • Internal style sheet is defined under head > style element and applies to any tagged elements on the page.
  • External style sheet is commonly preferred and makes changes easier. Keep in mind separation of concerns, or separation of code into different files based on usage.
  • A CSS rule contains selector(s) and {declaration block}. The selector defines the element(s) or attribute(s) the declarations apply to. The declaration block contains the properties and their values.
  • Every HTML element is a box the contains margin (outside the element) > border > padding > content (the last three are part of the element).
  • The asterisk, or wildcard symbol, * means the style will apply to all elements.
  • Select multiple elements separated with a comma (example: header, footer). Keep in mind Do Not Repeat Yourself (DRY) code to reduce size and complexity of code.
  • Create a class selector by preceding the element with a period.
  • Refer to colors using semantics such as gray or blue, or by using hex code #.
  • The margin property indicates amount of space outside an element.
  • The padding property adds space around the content inside an element.
  • The box-shadow property adds adds a shadow effect around the element's frame and uses offset values of X and Y, blur and spread radius, and color.

Git

GitHub Flow:

  1. Create a branch. The default branch is called main and it should be the final, most updated version of the application. Complete additional work with feature branches. Features need a good name to indicate the purpose of it.
  2. Make changes. Commit and push changes to your branch often. Each commit is an isolated, complete change.
  3. Create a pull request (PR). Add descriptive title. The body should contain what changes were made, how the feature is supposed to function/what problem it solves, and how to start the application so it can be tested. Link any issues the changes addresses. Ask for feedback. Some repos require an approving review before pull requests can be merged.
  4. Address review comments.
  5. Merge your pull request once approved. May need to address merge conflicts.
  6. Delete your branch. Indicates work is complete and prevents anyone from accidentally using old branches. Pull request and commit history will not be deleted and you can always restore deleted branches or revert pull requests if needed.

Git Commands:

  • git must precede every Git statement.
  • git status: check current working branch.
  • git checkout -b branch-name: create new branch and switch to it.
  • git checkout branch-name: switch to existing branch.
  • git revert [SHA]: revert code to specific version defined by SHA.
  • git add -a: add changes to staged branch (or next commit).
  • git commit -m "commit message": create snapshot of staged changes and add commit message or description.
  • git pull origin branch-name: fetch and merge remote commits into local enviroment. It is important to always do a git pull before you start to work.
  • git push origin feature/feature-name: send local branch to remote GitHub repo. Push can create a new remote branch or merge changes to an existing branch.
